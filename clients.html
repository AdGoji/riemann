Clients send events to Riemann.

Riemann clients speak Protocol Buffers over TCP and UDP. They can submit events to Riemann and query the index.

Tools

A Ruby gem with little daemons to monitor key systems.

Ruby

The Riemann Ruby client is my threadsafe, auto-retrying reference implementation, supporting both TCP and UDP. It also provides utility classes to help manage service state.

Python

Bernhard is Banjiewen's Python client. You can install it with pip install bernhard

Gleicon has also written a Python client which supports submitting events over TCP and UDP, and querying the index.

Java

The Java client provides a thread-safe TCP client. It's used by Riemann internally. I'm still working on getting this packaged up properly in clojars/maven central.

{% highlight xml %} boundary-site http://maven.boundary.com/artifactory/repo com.aphyr riemann-java-client 0.0.1 {% endhighlight %}

Node.js

Derek Perez is working on a Node.js client.

Erlang

Reify Health has published Zeta, an Erlang client.

Clojure

Riemann comes with a built-in Clojure client. It's thread-safe, stateful, and synchronized. Does not support UDP yet. You can require it as a dependency from Clojars. See riemann-bench for an example application using the client.
