Ready? Let's go!

For this tutorial, you'll need a *nix system, Java, and Ruby.

Install Reimann

wget http://aphyr.com/reimann/reimann-0.0.3-SNAPSHOT.tar.bz2
tar xvfj reimann-0.0.3-SNAPSHOT.tar.bz2
cd reimann-0.0.3-SNAPSHOT

Start the server

bin/reiman etc/reimann.config

Reimann is now listening for events. Install the Ruby client, tools, and dashboard:

gem install reimann-client reimann-tools reimann-dash

Start the dashboard. If reimann-dash isn't in your path, check your rubygems bin directory.

reimann-dash

Point your web browser to http://localhost:4567/. You should see a few empty boxes; that's because there are no events in the index yet. Let's send some:

reimann-health

The reimann-health daemon is a little Ruby program that submits events about the state of your CPU, memory, load, and disks to a Reimann server. If you switch back to the dashboard, you'll see your local host's state appear under Health. The Problems list shows any states that aren't "ok", so if you're running low on disk or ram they'll appear there. You can hover over a state, like CPU, to see its description.

The Ruby client

Let's send some of our own states.

$ irb -r reimann/client
ruby-1.9.3 :001 > r = Reimann::Client.new
 => #<Reimann::Client ... >

We can send events with <<. For example, let's log an HTTP request.

ruby-1.9.3 :002 > r << { 
  host: "www1", 
  service: "http req", 
  metric: 2.53, 
  state: "critical", 
  description: "Request took 2.53 seconds.", 
  tags: ["http"]
}

On the dashboard, critical events (like the one we just submitted) show up in red. All of these fields are optional, by the way. The ruby client will assume your events come from the local hostname unless you pass host: nil.

Now let's ask for all events that have a service beginning with "http".

ruby-1.9.3 :003 > r['service = "http%"']
[<Reimann::Event time: 1330041937, state: "critical", service: "http req", host: "www1", 
description: "Request took 2.53 seconds.", tags: ["http"], ttl: 300.0, metric_f: 2.5299999713897705>]

There's the event we submitted. You could send these events from Rack middleware to log every request to your app, or track exceptions. Reimann can be configured to calculate rates, averages, min/max, distributions, and more. To get started, read the configuration guide.
