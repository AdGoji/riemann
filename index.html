Reimann monitors distributed systems.

Reimann aggregates events from your servers and applications with a powerful stream processing language. Send an email for every exception raised by your code. Track the latency distribution of your web app. See the top processes on any host, by memory and CPU. Combine statistics from every Riak node in your cluster and forward to Graphite. Send alerts when a key process fails to check in. Know how many users signed up right this second.

Reimann provides low-latency, transient shared state for systems with many moving parts.

Instant overview

A small, extendable Sinatra app shows your system at a glance. Instantly identify hotspots, down services, and unbalanced loads.

Email alerts

(rollup 5 3600
            (email "dev@startup.com", "ops@startup.com"))

Reimann can tell you as much or as little as you want. Throttle or roll up multiple events into a single messages. Get emails about exceptions in your code, provider downtime, or latency spikes.

Powerful stream primitives

(where (and (service "api") 
                (not (tagged "transient")))
      graph
      index
      (changed-state 
        (email "5551231234@vtext.com")))

Reimann streams are just Clojure functions. You can use dozens of built-in streams for combining, filtering, altering, and forwarding events, or write your own.

Graph everything

(graphite {:host "graphs.pdx"})

Like Statsd, Reimann can forward any event stream to Graphite.

Query states

state = "error rate" and (not host =~ "www.%")

Search the Reimann index with a small query language. Clients can monitor each other, generate reports, or produce dashboards.

Plays well with others

(where (tagged "important") 
  (forward overview-server))

Forward events transparently to other Reimann servers or monitoring tools. Any Java or Clojure client will do.

Simple clients

r = Reimann::Client.new
r << {service: "www", state: "down", metric: 10000}
r['state = "down"'] 
# => [#<Reimann::Event @service="www" ... >]

Reimann speaks Protocol Buffers over TCP and UDP for a compact, portable, and fast wire protocol. See the Ruby client as a guide.
